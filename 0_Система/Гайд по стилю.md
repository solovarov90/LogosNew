# 📋 Гайд по оформлению Базы Знаний

## 1. Структура файла (Стандарт)
Каждый файл начинается с YAML-блока. Обязательно используем один широкий макро-тег для раскраски в Графе, а узкие подтеги (напр. `#маркетинг/UX`) оставляем для детального поиска.

```markdown
---
type: концепт | новости | синтез | материалы
status: черновик | исследование | база
source: [[...]]
tags: 
  - "#психология"  <-- ОБЯЗАТЕЛЬНЫЙ МАКРО-ТЕГ ДЛЯ ГРАФА
  - "#маркетинг/нейромаркетинг" <-- узкий тег для поиска
---
```

## 2. Макро-теги (Для цветовой фильтрации Графа)
При создании новых заметок всегда добавляем один из 5 базовых макро-тегов, чтобы файл сразу попал в правильную цветовую зону на графе:
- `#психология` (поведенческая экономика, нейромаркетинг, UX, Канеман, Чалдини)
- `#контент` (копирайтинг, редактура, SMM, виральность)
- `#стратегия` (Голубые океаны, позиционирование, стартапы, конкуренты)
- `#удержание` (геймификация, лояльность, CJM, отток, онбординг)
- `#продажи` (воронки, лидогенерация, B2B, SPIN, виральность)

## 3. Настройки Графа Obsidian (Скопируй и вставь в Группы)
Копировать нужно строго эти строчки (без кавычек и решеток). Приоритет сверху вниз (широкие теги должны быть выше узких папок).

1. 🟢 Психология: `tag:психология`
2. 🔴 Контент: `tag:контент`
3. 🟣 Стратегия: `tag:стратегия`
4. 🟠 Удержание: `tag:удержание`
5. 🔵 Продажи: `tag:продажи`
6. 🟡 Ядро системы: `tag:система OR tag:ядро`

## 4. По стадиям готовности (Внутренние теги):
- `#исследование` (сырые материалы)
- `#кристаллизация` (чистые концепты)
- `#синтез` (готовые болванки для ИИ)
