# Правила и Логика Автоворонки Smartposting 🦾

Этот файл определяет системные правила генерации сообщений для рассылки в Telegram-боте.

## 🛠 Технический стандарт
- **Формат:** Строго HTML.
- **Переменные:** `{name}`, `{username}`, `{date}`, `{time}`.
- **Ссылки:** Использование `<a href="...">...</a>` для кнопок и переходов.

## 🎯 Архитектура закрытия (CTA)
В каждом сообщении должно быть целевое действие. Варианты:
1.  **Обучение:** Ссылка с промптом `/start learn` (выдает 3 урока).
2.  **Поддержка:** Ссылка на саппорта (если есть вопросы).
3.  **Канал:** Ссылка на основной канал с «ништяками».
4.  **Гайды:** Ссылки на сообщения в канале с Гайдом №1 и №2.

## 🧠 Смысловые акценты (Сдвиги парадигмы)
- **ИИ-текст vs Человек:** Ломаем убеждение о «пластмассовости» через плейсхолдеры для отзывов и статистики (аномальные дочитывания лонгридов).
- **Личность Архитектора:** Пётр Соловаров (3300+ сабов за 2.5 мес, опыт работы с топ-школами).
- **Дефицит:** Только 10 слотов из-за ограничений официального API Threads (всего 100 аккаунтов на тест).

## 📊 Плейсхолдеры для контента
При генерации постов оставлять маркеры для вставки данных внешней нейронкой:
- `[PLACEHOLDER: STATS_READ_RATE]` — статистика дочитываний.
- `[PLACEHOLDER: FEEDBACK_SCREEN]` — скриншоты восторженных отзывов.
- `[PLACEHOLDER: FOLLOWER_COUNT]` — динамика роста канала.
- `[PLACEHOLDER: API_SLOTS_LEFT]` — количество оставшихся мест.

## 📅 Ритмика рассылки
- **Дни 1-6 (Прогрев):** 1-2 сообщения в день.
- **Дни продажи:** 4-6 сообщений (эффект обратного отсчета).
