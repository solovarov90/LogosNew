# ПРОТОКОЛ: ЗАПУСК НОВОГО ГАЙДА (СХЕМА ЗАПУСКА) 🧬🏗️🧿

Этот документ описывает стандартную схему запуска нового лид-магнита (гайда) в экосистеме Smartposting. Используется каждый раз, когда Архитектор готовит новый материал.

### 1. Текстовые компоненты (Asset List)
Для каждого запуска необходимо подготовить:
1.  **Сообщение в боте (Выдача):** Текст, который пользователь получает сразу после нажатия кнопки/ввода промпта. Содержит прямую ссылку на гайд и краткий "Why now?" тезис.
2.  **Рассылка по базе (Анонс):** Сообщение для тех, кто уже в боте. Задача — перевести их на канал, где лежит основной пост.
3.  **Пост-концепт (Канал):** Центральный смысловой пост на канале. Поздравление, вскрытие новой боли/задачи и призыв забрать гайд в боте по кодовому слову.

### 2. Структура выдачи гайда (Bot Response)
- Приветствие + Подтверждение получения.
- Ссылка на файл (Drive/PDF).
- **CTA:** Что сделать сразу после прочтения (например, "Напиши мне голос в чат").

### 3. Логика Рассылки (Broadcast)
- **Триггер:** "Мы обновили Экзоскелет".
- **Интрига:** Какой конкретно результат дает этот гайд (например, "Первые 100 сабов за 24 часа").
- **Переход:** Ссылка на пост в канале (Social Proof).

### 4. Пост-концепт (Channel Post)
- **Hook:** Поздравление новых клиентов/подписчиков.
- **Core Concept:** Разбор главной идеи гайда (например, "Точка невозврата — 100 подписчиков").
- **Instruction:** Как забрать (Бот + Промпт).

---
**Запись в карту:** При следующем запуске Архитектор дает ссылку на гайд и свои пожелания, а Логос генерирует эти 3 типа текстов согласно протоколу.
