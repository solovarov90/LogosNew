# SmartPosting — паспорт и типы рассылок

> **ГЛАВНОЕ:** рассылки и цепочки **создаются и реально отправляются в приложении SmartPosting**
> через бот-API. Черновик в `3_Запуски/` — это подготовка/архив; фактическая отправка
> по базе бота — только через API (см. протокол «РАССЫЛКА ЧЕРЕЗ SMARTPOSTING» в `AGENTS.md`).

## Как отправить рассылку (рабочий цикл)
Доступы уже в окружении: `SMARTPOSTING_API_URL`, `SMARTPOSTING_BOT_KEY`.
1. Собрать план (1 шаг = 1 рассылка; цепочка = несколько шагов со `scheduledAt`), показать в чате, **дождаться «да»**.
2. Создать задачу: `POST $SMARTPOSTING_API_URL/api/bot/tasks` (заголовок `Authorization: Bearer $SMARTPOSTING_BOT_KEY`), тело `{type,title,chatId,plan[]}`. Вернётся `task._id`.
3. Запустить: `POST $SMARTPOSTING_API_URL/api/bot/tasks/<ID>/execute`.
4. Опрашивать `GET $SMARTPOSTING_API_URL/api/bot/tasks/<ID>` до `completed`/`failed`, отчитаться в чат (`report.total/sent/failed`).
Статусы задач видны во вкладке **«Бот»** в приложении SmartPosting.

Формат шага плана: `{ order, summary, text(HTML), mediaUrls?, mediaType?, button?{text,url}, segment?, scheduledAt? }`.

## Что такое SmartPosting
AI-платформа для блогеров/SMM: генерация контента, планировщик, анализатор, Telegram-бот для публикации. Тарифы: free / start / pro / vip. Рассылки уходят по **базе подключённого бота** (для админа — база бота SmartPosting; для VIP — база его собственного бота).

## Типы рассылок (для сборки плана)

### 1. Анонс
Хук → суть → один CTA (кнопка). Сегмент по умолчанию `all`.

### 2. Приглашение на эфир
Дата/время, тема, польза, кнопка записи. Хорошо цепочкой: анонс за 2–3 дня → напоминание в день → «мы в эфире». Сегменты `all`/`active`.

### 3. Скидка / оффер
Выгода, дедлайн, ограничение, кнопка оплаты. На тёплых: `registered`/`active`/`clients`. Цепочка: оффер → «осталось N» → «последний день».

### 4. Прогрев (цепочка)
3–5 сообщений: боль → история/кейс → решение → оффер → дедлайн. Каждый шаг — свой `scheduledAt`. Тип задачи `broadcast_chain`.

### 5. Реактивация
Для «спящих» — вернуть вниманием/бонусом. Сегмент `leads`/custom.

## Сегменты (поле `segment`)
- `all` — вся база бота
- `leads` — без email/без регистрации
- `registered` — зарегистрированы, без активности
- `active` — есть посты/активность
- `clients` — платные (pro/vip)

## Правила текста
- Живой тон, без AI-штампов. Одна мысль — один CTA.
- Бот = HTML (`<b>`, `<a href>`); Канал = MarkdownV2 (по контентным протоколам).
- `{name}` подставит имя получателя.
